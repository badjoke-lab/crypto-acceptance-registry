# Roadmap

## Phase 0: Definition
- fix product scope
- fix classification rules
- fix evidence rules
- define legacy import boundary

## Phase 1: Legacy extraction
- export candidate merchant records from legacy CPM data
- export payment_accepts, socials, and verification metadata
- produce normalized intermediate dataset

## Phase 2: Classification pipeline
- rule-based first-pass classifier
- confidence scoring
- evidence attachment and normalization
- review queue generation

## Phase 3: Review workflow
- review queue UI or review-ready data output
- approve/fix/reject flow for classification outcomes
- merge reviewed outputs into canonical dataset

## Phase 4: Product v1
- searchable merchant list
- merchant detail page
- minimal stats
- evidence-first display

## Phase 5: Expansion
- broader evidence collection
- better review tooling
- map or geographic layer only if justified by the data
